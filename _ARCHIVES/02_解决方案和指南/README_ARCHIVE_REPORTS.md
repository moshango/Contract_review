## 📌 文档命名规范

为了保持良好的组织，未来新增报告请遵循以下命名规范：

### 模式1：主题_类型_日期.md
```
CHATGPT_INTEGRATION_SUMMARY.md
PARSE_ANNOTATE_WORKFLOW_REPORT.md
```

### 模式2：主题_说明.md
```
DEBUG_AND_FIX_GUIDE.md
PARSERESULTCACHE_IMPLEMENTATION_REPORT.md
```

### 推荐命名
- 使用大写字母和下划线
- 文件名简洁明了，能准确反映内容
- 避免使用中文字符（便于跨平台使用）
